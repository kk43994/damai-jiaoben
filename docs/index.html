📥 完整安装教程

从零开始，手把手教你安装所有环境

第1步：安装Python环境

⏱️ 预计耗时：5-10分钟

📌 什么是Python？

Python是一种编程语言，我们的程序是用Python写的，所以电脑上必须先安装Python才能运行这个项目。就像你要看Word文档必须先安装Microsoft Office一样。

1.1 下载Python安装包

第1步：打开浏览器（推荐使用Chrome或Edge）

第2步：在地址栏输入以下网址并回车：

https://www.python.org/downloads/

第3步：进入网站后，你会看到一个黄色的大按钮，上面写着"Download Python 3.x.x"

  • 这个3.x.x就是版本号，比如3.11.9
  • 重要：确保版本号是3.11或更高（3.12、3.13都可以）
  • 不要下载3.10或更低版本，可能会不兼容

第4步：点击黄色按钮，开始下载

  • 文件大小约25MB
  • 下载时间根据网速，一般1-5分钟
  • 文件名类似：python-3.11.9-amd64.exe
  • 下载完成后，文件通常在"下载"文件夹中

1.2 安装Python

第1步：找到刚才下载的.exe文件，双击打开

⚠️ 极其重要的一步！请仔细阅读！

在安装界面的最下方，你会看到两个复选框，必须勾选第一个：

☑️ Add Python to PATH    ← 这个必须勾选！

❓ 为什么这一步这么重要？

如果不勾选这个选项，电脑将无法识别Python命令，后续所有步骤都会失败。就像你买了一辆车但没有上牌照，无法上路行驶。

💡 如何确认已勾选？

复选框里应该有一个蓝色的勾✓，如果是空白的就点一下它。

第2步：勾选"Add Python to PATH"后，点击"Install Now"（立即安装）

  • 不要点击"Customize installation"（自定义安装），除非你很懂电脑
  • 安装过程约3-5分钟
  • 期间可能会弹出"用户账户控制"窗口，点击"是"

第3步：等待安装完成

  • 你会看到进度条，显示"Installing..."（正在安装）
  • 安装完成后会显示"Setup was successful"（安装成功）
  • 点击"Close"（关闭）按钮

1.3 验证Python是否安装成功

💡 什么是验证？为什么要验证？

验证就是检查Python是否正确安装。就像你买了新手机，要开机看看能不能用一样。

📝 如何打开命令提示符（CMD）？

方法1（推荐）：

  1. 按键盘上的Windows键（通常在Ctrl和Alt之间，有个Windows图标）
  2. 直接输入cmd三个字母
  3. 你会看到"命令提示符"这个应用，点击它

方法2：

  1. 同时按Windows键 + R
  2. 在弹出的"运行"窗口中输入cmd
  3. 按回车键

方法3：

  1. 点击开始菜单（屏幕左下角的Windows图标）
  2. 找到"Windows系统"文件夹
  3. 点击展开，找到"命令提示符"

✅ 打开CMD后会看到什么？

一个黑色的窗口，里面有一些白色的文字，类似这样：

C:\Users\你的用户名>

这个黑色窗口就是"命令提示符"，后面简称CMD。

🧪 开始验证

第1步：在CMD窗口中，光标后面输入以下命令（不要多打空格）：

python --version

⚠️ 输入注意事项

  • python后面有两个减号--，不是一个
  • --和version之间有一个空格
  • 全部都是小写英文字母
  • 输入完成后按回车键

第2步：查看结果

✅ 成功的情况

如果安装成功，CMD会显示：

Python 3.11.9

（数字可能略有不同，比如3.11.8、3.12.0等，只要是3.11以上就可以）

恭喜！Python已经安装成功！可以进行下一步了。

❌ 失败的情况

如果显示以下任何一种错误信息：

'python' 不是内部或外部命令，也不是可运行的程序或批处理文件。

或者英文版：

'python' is not recognized as an internal or external command

🔧 解决方法

原因：你在安装时没有勾选"Add Python to PATH"

解决办法：

  1. 卸载刚才安装的Python（在"控制面板"-"程序和功能"中找到Python，右键卸载）
  2. 重新下载Python安装包
  3. 这次一定要勾选"Add Python to PATH"
  4. 重新安装
  5. 重新打开一个新的CMD窗口验证（必须是新的，旧的不行）

第2步：安装Node.js环境

📌 什么是Node.js？

Node.js是一个运行环境，Appium需要它才能工作。

💡 用生活场景打比方：
  • Appium = 一辆汽车（自动化工具）
  • Node.js = 汽油（没有汽油车开不了）
  • 我们的项目 = 驾驶员（控制汽车）

简单说：没有Node.js，Appium就无法运行。

2.1 下载Node.js安装包

第1步：打开浏览器

第2步：在地址栏输入Node.js官网地址：

https://nodejs.org/

第3步：进入网站后，你会看到两个绿色的大按钮

  • 右边的写着"Current"（最新版）

✅ 选择哪个版本？

点击左边的"LTS"版本！

LTS = Long Term Support（长期支持版），更稳定，不容易出问题

版本号通常是20.x.x或22.x.x（数字可能不同）

第4步：点击LTS按钮，下载会自动开始

  • 文件大小约30MB
  • 下载时间约2-5分钟

第2步：可能会弹出"用户账户控制"窗口

  • 点击"是"或"允许"

第3步：进入安装向导

✅ 安装很简单：一路点"Next"就行！

你会看到大约5-6个界面，每个都点"Next"（下一步）按钮。

  • 不需要勾选"Launch..."（启动...），直接点Finish

2.3 验证Node.js是否安装成功

💡 为什么要验证两个命令？

Node.js安装包含两个工具：

  • node = Node.js运行环境（汽车的发动机）
  • npm = 包管理工具（用来安装其他软件的工具，类似应用商店）

两个都要验证，确保都安装成功了。

🧪 开始验证

第1步：打开CMD命令提示符

  • 按Windows键
  • 输入cmd
  • 点击"命令提示符"

第2步：验证node（在CMD中输入以下命令，然后按回车）

node --version
✅ 成功的情况

如果显示：

v20.11.1

（数字可能不同，只要有v开头就成功了）

恭喜！Node.js安装成功！

第3步：验证npm（在CMD中输入以下命令，然后按回车）

npm --version
✅ 成功的情况

如果显示：

10.2.4

（数字可能不同，只要显示数字就成功了）

太好了！npm也安装成功了！

❌ 如果出现错误

显示类似：

'node' 不是内部或外部命令

或

'npm' 不是内部或外部命令
🔧 解决方法
  1. 先关闭CMD窗口
  2. 重启电脑（很重要！）
  3. 重启后重新打开CMD
  4. 再次输入node --version和npm --version检查
  5. 如果还不行，可能需要卸载后重新安装Node.js

第3步：安装Android SDK

⏱️ 预计耗时：10-15分钟

📌 什么是Android SDK？

SDK = Software Development Kit（软件开发工具包）

Android SDK包含了ADB工具，ADB是用来连接和控制Android设备的。

💡 用生活场景打比方：
  • 你的手机/云手机 = 一台电视机
  • ADB工具 = 电视遥控器
  • 我们的程序 = 你的手（操作遥控器）

简单说：没有ADB，电脑就无法连接和控制手机。

3.1 下载Android SDK Platform Tools

第1步：打开浏览器

第2步：在地址栏输入以下网址：

https://developer.android.com/studio/releases/platform-tools

⚠️ 如果网页打不开或很慢

Google的网站在国内可能比较慢，可以尝试：

  1. 多刷新几次
  2. 换个时间段访问
  3. 或者直接使用下面的国内镜像地址（更快）：
https://www.androiddevtools.cn/

进入后，找到"SDK Platform Tools"，下载Windows版本

第3步：在官网页面向下滚动，找到"Downloads"（下载）部分

第4步：找到"SDK Platform Tools for Windows"

  • 会看到一个链接，写着"Download SDK Platform-Tools for Windows"
  • 点击这个链接

第5步：弹出协议对话框

  • 会弹出一个窗口，显示使用条款
  • 勾选底部的"I have read and agree..."（我已阅读并同意...）
  • 点击下方的蓝色按钮"Download Android SDK Platform-Tools for Windows"

第6步：开始下载

  • 文件大小约10MB
  • 下载时间约1-3分钟
  • 文件名：platform-tools-latest-windows.zip
  • 下载完成后在"下载"文件夹中

3.2 解压SDK文件

第1步：找到刚才下载的platform-tools-latest-windows.zip文件

第2步：决定解压到哪里

✅ 推荐的解压位置

建议解压到C盘根目录，路径更短，更容易记住：

C:\android-sdk\

为什么选这里？

  • 路径简单，不容易出错
  • 后面配置环境变量时更方便
  • 不会被误删（不在桌面或下载文件夹）

第3步：创建android-sdk文件夹

  1. 打开"此电脑"或"我的电脑"
  2. 双击进入C盘
  3. 在C盘空白处右键 → 新建 → 文件夹
  4. 文件夹命名为：android-sdk（全部小写，中间有减号）

第4步：解压ZIP文件

  1. 回到"下载"文件夹，找到platform-tools-latest-windows.zip
  2. 右键点击这个ZIP文件
  3. 选择"全部提取..."或"解压缩到..."
  4. 在弹出的窗口中，点击"浏览"按钮
  5. 找到并选择刚才创建的C:\android-sdk\文件夹
  6. 点击"提取"或"确定"

第5步：验证解压结果

打开C:\android-sdk\文件夹，你应该看到一个platform-tools文件夹

打开platform-tools文件夹，里面应该有很多文件，其中包括adb.exe

完整路径应该是：C:\android-sdk\platform-tools\adb.exe

3.3 配置环境变量

❓ 什么是环境变量？为什么要配置？

通俗解释：环境变量就像给电脑建一个"快捷方式列表"

配置后，你在CMD里输入adb，电脑就知道去C:\android-sdk\platform-tools\找这个程序

如果不配置，电脑会说"找不到adb"

📝 超详细配置步骤

第1步：打开"此电脑"

  • 在桌面上找到"此电脑"图标（或"我的电脑"）
  • 如果桌面上没有，按Windows键，输入"此电脑"

第2步：打开系统属性

  1. 右键点击"此电脑"图标
  2. 在弹出菜单中选择"属性"
  3. 会打开一个窗口，显示你的电脑信息

第3步：进入高级系统设置

⚠️ 这一步根据Windows版本不同，可能有两种情况

情况A（Windows 11或新版Win10）：

  1. 在打开的窗口右侧，找到"高级系统设置"链接
  2. 点击它

情况B（老版Windows 10）：

  1. 在左侧菜单，找到"高级系统设置"
  2. 点击它

第4步：打开环境变量

  1. 会弹出"系统属性"窗口
  2. 确认顶部标签页选中的是"高级"
  3. 在窗口底部，找到"环境变量(N)..."按钮
  4. 点击这个按钮

第5步：编辑Path变量

  1. 会弹出"环境变量"窗口，分为上下两部分
  2. 看下面那部分，标题是"系统变量(S)"
  3. 在列表中找到变量名为"Path"的那一行（可能需要滚动查找）
  4. 单击选中"Path"这一行（变成蓝色高亮）
  5. 点击下方的"编辑(I)..."按钮

第6步：添加新路径

  1. 会弹出"编辑环境变量"窗口
  2. 点击右侧的"新建(N)"按钮
  3. 会出现一个新的空行，光标在闪烁
  4. 在这个空行中输入（注意大小写和斜杠方向）：
C:\android-sdk\platform-tools

⚠️ 输入注意事项

  • 必须是反斜杠 \，不是正斜杠 /
  • 路径必须和你实际解压的位置完全一致
  • 如果你解压到了其他位置（比如D盘），路径就要相应改变
  • 不要在路径末尾加\adb.exe，只到platform-tools文件夹即可

第7步：保存设置

  1. 点击"编辑环境变量"窗口的"确定"按钮
  2. 点击"环境变量"窗口的"确定"按钮
  3. 点击"系统属性"窗口的"确定"按钮
  4. 重要：如果有CMD窗口打开着，必须关闭后重新打开才能生效

3.4 验证安装是否成功

第1步：打开新的CMD窗口

  • 按Windows键
  • 输入cmd
  • 点击"命令提示符"

第2步：输入验证命令

✅ 成功的情况

如果显示类似这样的信息：

Android Debug Bridge version 1.0.41

Installed as C:\android-sdk\platform-tools\adb.exe

（版本号可能不同，只要有"Android Debug Bridge version"就成功了）

❌ 如果出现错误

显示：

'adb' 不是内部或外部命令

🔧 解决方法

  1. 检查路径：确认C:\android-sdk\platform-tools\adb.exe文件存在
  2. 检查环境变量：重新打开环境变量设置，确认Path中有C:\android-sdk\platform-tools

4.1 安装Appium主程序

⚠️ 前提条件检查

开始之前，确保你已经完成了：

💡 打个比方：
  • Appium = 汽车的车身

📖 命令解释

  • install = 安装
  • uiautomator2 = 驱动的名称

第3步：按回车键，开始安装

第4步：等待安装完成

❌ 如果安装失败

通常是网络问题，解决方法：

  1. 检查网络连接
  2. 多试几次
  3. 使用国内镜像：

2. 设备连接

📌 什么是"设备连接"？

简单理解：就是让你的电脑和红手指云手机"建立联系"，这样电脑才能控制云手机。

比喻：就像你的手机要连接蓝牙耳机，必须先"配对"一样。

2.1 准备工作

第1步：确保红手指云手机已开启

  • 登录红手指官网：https://www.redfinger.com/
  • 在控制台中，你的云手机状态应该显示为"运行中"或"在线"
  • 如果显示"已停止"或"离线"，点击"启动"按钮
  • 等待约1-2分钟，直到云手机完全启动

第2步：确认ADB端口号

  • 在红手指控制台，点击你的云手机
  • 找到"查看详情"或"ADB连接"按钮
  • 你会看到一个5位数字的端口号，比如：12345
  • 记住这个端口号，后面会用到

第3步：检查配置文件中的端口号

  • 打开项目文件夹，找到damai_appium/config.jsonc文件
  • 右键 → 用记事本打开（或任何文本编辑器）
  • 找到这一行："adb_port": "12345",
  • 把12345改成你的实际端口号
  • 保存文件（Ctrl + S）

2.2 开始连接

第1步：在GUI界面中找到"连接设备"按钮

  • 这个按钮通常在GUI的左上角
  • 按钮上写着"连接设备"或"Connect Device"

第2步：点击"连接设备"按钮

  • 点击后，按钮可能会变灰（表示正在连接）
  • 等待3-10秒
  • 期间GUI会自动读取配置文件中的ADB端口

第3步：查看连接结果

2.3 判断连接是否成功

✅ 成功的情况

在GUI的日志区域（通常是窗口下方的黑色框），你会看到绿色的文字：

[SUCCESS] 设备连接成功

或者：

[SUCCESS] Device connected successfully
还可能看到以下信息：
  • [INFO] 设备型号: xxx
  • [INFO] Android版本: x.x
  • [INFO] 屏幕分辨率: 1920x1080

恭喜！连接成功，可以进行下一步了！

❌ 失败的情况

如果看到红色的错误信息，比如：

[ERROR] 设备连接失败
[ERROR] Cannot connect to device
[ERROR] ADB connection failed
🔧 解决方法一：使用连接急救箱（推荐）

这是最简单的方法，让程序自动诊断并修复问题：

  1. 在GUI界面找到"🏥 连接急救箱"按钮（通常在"连接设备"旁边）
  2. 点击这个按钮
  3. 会弹出一个新窗口
  4. 按照窗口中的提示操作：
    • 如果提示输入端口号，输入你的红手指ADB端口
    • 点击"自动诊断"或"一键修复"按钮
    • 等待程序自动检测和修复问题
  5. 修复完成后，重新点击"连接设备"
🔧 解决方法二：手动检查

如果急救箱也无法解决，按以下步骤逐一检查：

  1. 检查红手指云手机是否在线
    • 登录红手指控制台
    • 确认云手机状态是"运行中"
    • 如果不是，点击"启动"
  2. 检查ADB端口号是否正确
    • 红手指控制台显示的端口号
    • 配置文件config.jsonc中的端口号
    • 这两个端口号必须完全一致
  3. 检查网络连接
    • 确保电脑能上网
    • 尝试打开其他网页，看网络是否正常
  4. 重启Appium服务器
    • 关闭GUI程序
    • 重新运行一键启动.bat

错误信息 可能原因 解决方法

❓ 常见问题

Q1: 提示"adb不是内部或外部命令"怎么办？

A:

原因：Android SDK未安装或环境变量未配置

解决方法：

  1. 确认已安装Android SDK Platform Tools
  2. 配置环境变量PATH，添加platform-tools目录
  3. 重新打开命令提示符测试
  4. 如果使用一键启动脚本，确保通过脚本启动

Q2: 连接设备失败，显示"device offline"？

A:

原因：红手指云手机未开启或ADB端口错误

解决方法：

  1. 确认红手指云手机已开启
  2. 检查ADB端口号是否正确
  3. 使用"连接急救箱"功能自动诊断
  4. 手动执行：adb disconnect后重新连接

Q3: Appium启动失败，提示端口被占用？

A:

原因：4723端口已被其他程序占用

解决方法：

  1. 关闭所有Appium实例
  2. 运行：taskkill /F /IM node.exe /T（Windows）
  3. 重新启动Appium

Q4: pip install失败，提示网络超时？

A:

原因：网络问题或pypi.org被限速

解决方法：

# 使用国内镜像源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 或者配置永久镜像源
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple

Q5: 运行中途卡住不动了？

A:

可能原因：

  • WebDriver连接中断
  • 页面加载超时
  • App界面发生变化

解决方法：

  1. 查看日志输出，确认卡在哪一步
  2. 点击"停止流程"
  3. 使用"连接急救箱"修复连接
  4. 重新开始抢票

Q6: 搜索不到演出？

A:

原因：关键词不准确或演出未上线

解决方法：

  1. 确认关键词与大麦App中的演出名称一致
  2. 尝试使用简短的关键词（如艺人名字）
  3. 检查演出是否已在大麦App上线
  4. 手动在App中搜索，确认演出存在

Q7: 提示"找不到指定城市"？

A:

原因：城市名称不匹配或演出不在该城市

解决方法：

  1. 确认城市名称完全匹配（如"北京"不能写成"北京市"）
  2. 检查该演出是否在目标城市有场次
  3. 先手动搜索确认城市列表

Q8: PaddleOCR识别失败？

A:

原因：模型未正确安装或设备分辨率不支持

解决方法：

  1. 确认已安装paddleocr和paddlepaddle
  2. 首次运行会自动下载模型，需要耐心等待
  3. 检查设备截图是否清晰
  4. 必要时在config.jsonc中设置"enable_ocr": false
